SPO: off

Connect to your tenant

InactiveGuests runs entirely in your browser. It signs you in with Microsoft (PKCE) and queries Microsoft Graph as you. No backend, no data leaves your browser.

Required delegated permissions:

  • User.Read.All – list guest users
  • AuditLog.Read.All – read sign-in activity
  • Directory.Read.All – list deleted users
  • User.Read – sign-in identity
  • User.ReadWrite.All + Directory.AccessAsUser.All (only when you disable, delete or restore)
  • Sites.ReadWrite.All (only when SPO governance integration is enabled)

To act on guests you also need User Administrator or Global Administrator in Entra.

    About InactiveGuests
    
      
    InactiveGuests surfaces dormant guest accounts in your Entra ID tenant so you can disable, delete or restore them in bulk. It is designed to work alongside the Automated Guest User Lifecycle Management runbook — sharing the same exclusion list and audit log so interactive and automated actions are visible in one place.
    

      
    SharePoint integration
    
      
    Configure your SharePoint site URL and list names in the Settings tab. Once enabled:
    
      
      
        
    • Guests on the GuestExclusionList are highlighted 🔒 and blocked from bulk actions.
      • 
        
    • Every interactive Disable / Enable / Delete / Restore is written to GuestAuditLog with Action=Interactive, the admin's UPN, and a timestamp — so it appears alongside runbook actions in the same list.
      • 
      
    

      
    What it can do
    
      
      
        
    • List every guest with creation date, redemption state, enabled status, and last sign-in (interactive + non-interactive).
      • 
        
    • Filter on inactivity threshold, account age, redemption state, enabled state, and sign-in history.
      • 
        
    • Bulk disable, re-enable, or delete selected guests with an explicit confirmation dialog.
      • 
        
    • Export filtered or selected guests to XLSX or CSV.
      • 
        
    • Browse the recycle bin and restore any guest within the 30-day window.
      • 
      
    

      
    How sign-in activity works
    
      
    Takes the most recent of lastSignInDateTime and lastNonInteractiveSignInDateTime. If neither exists, falls back to createdDateTime so new guests don't appear "inactive forever". Requires Entra ID P1 or P2 for sign-in data to be available.
    

      
    Permissions
    
      
    Read-only browsing: User.Read.All, AuditLog.Read.All, Directory.Read.All, User.Read.
    
      Write actions: adds User.ReadWrite.All, Directory.AccessAsUser.All.
    
      SPO integration: adds Sites.ReadWrite.All.
